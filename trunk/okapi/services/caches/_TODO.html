Consumer: required Token: optional
WRTODO
WRTODO Get a list of geocaches. Either area, point, wp name, or owner are required. Results are sorted by distance from the center point. If not center point is provided, then they are sorted by the distance from the center of the bbox. If neither of those are provided, then they are sorted by oxcode.
status_in optional

Default value: 1

Pipe-separated list of status codes. Only caches matching any of the given statuses will included in the response. Possible status codes are:

  • 1 - Ready to search,
  • 2 - Temporarily unavailable,
  • 3 - Archived.
format optional

Default value: json

One of the following: json, xml or gpx. Format in which you'd like the results to be returned.
area optional

If given, limits the geocaches returned to those being inside the specified bounding box. Caches will be sorted by the distance from the center of this area.

This parameter must be given in the following format:
south_latitude|west_longitude|north_latitude|east_longitude (the edges of the bounding area). Use positive numbers for north latitude and east longitude and negitive numbers of south latitude and west longitude.
point optional If given, the method will return a list of geocaches closest to the specified point sorted by the distance from that point.
wpts optional Pipe-separated list of waypoint names. If given, the list of returned caches will be limited to the waypoints given.
owners optional Pipe-separated list of user IDs. If given, the list of returned caches will be limited to those owned by the given users.
found_status optional

Default value: either

Notice: This parameter may be used only for requests signed with an Access Token.

Should be one of the following:

  • found_only - only caches found by the user will be returned,
  • unfound_only - only caches not found by the user will be returned,
  • either - all caches will be returned.
exclude_my_own optional

Default value: false

Notice: This parameter may be used only for requests signed with an Access Token.

Boolean. If set to true, caches which the user is an owner of will not be included in the result.
modified_since optional

A datetime value in a format of "yyyy-mm-dd hh:mm:ss". If given, the list of returned geocaches will be limited to those which were created or modified since that date.
dist optional

Valid along the point parameter. This is the radius within which to search for caches. All caches which are more than dist kilimeters away from the center point will not be included in the results.
desc optional

Default value: false

Boolean. Tells us whether to include cache descriptions or not. Currently, must be set to false when format is set to gpx.
min_recommendations optional

Integer. If N given, the result will contain only those caches which have at least N recommendations.
terrain optional

Default value: 1-5

A string "X-Y", where X and Y are integers between 1 and 5, and X <= Y. Only caches with terrain rating between these numbers (inclusive) will be returned.
difficulty optional

Default value: 1-5

A string "X-Y", where X and Y are integers between 1 and 5, and X <= Y. Only caches with difficulty rating between these numbers (inclusive) will be returned.
size optional

Default value: 1-5

A string "X-Y", where X and Y are integers between 1 and 5, and X <= Y. Only caches of size rating between these numbers (inclusive) will be returned.
score optional

Default value: 1-5

A string "X-Y", where X and Y are integers between 1 and 5, and X <= Y. Only caches with an overall score rating between these numbers (inclusive) will be returned.
type optional

Pipe-separated list of cache type codes. If given, the results will be limited to the given cache types. Valid cache type codes currently include:

  • 1 - Other cache,
  • 2 - Traditional cache,
  • 3 - Multi-cache,
  • 4 - Virtual cache,
  • 5 - Webcam cache,
  • 6 - Event cache,
  • 7 - Quiz cache,
  • 8 - Moving cache,
  • 9 - Own cache.
